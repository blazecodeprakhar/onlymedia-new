# Contributing to OnlyMedia 🤝

Thank you for your interest in contributing to **OnlyMedia**! We welcome contributions of all kinds — bug fixes, new features, documentation improvements, and more.

---

## 🚀 Getting Started

### Prerequisites
- Node.js **18+**
- npm or yarn
- Git

### Setup

```bash
# 1. Fork the repository
# 2. Clone your fork
git clone https://github.com/YOUR_USERNAME/onlymedia-new.git
cd onlymedia-new

# 3. Install dependencies
npm install

# 4. Start the dev server
npm run dev
```

---

## 🔄 Development Workflow

1. **Create a branch** from `main`:
   ```bash
   git checkout -b feature/your-feature-name
   ```

2. **Make your changes** with clear, focused commits.

3. **Follow the commit convention**:
   ```
   feat:  new feature
   fix:   bug fix
   docs:  documentation update
   style: code style / formatting
   refactor: code refactoring
   ```

4. **Push your branch** and open a Pull Request:
   ```bash
   git push origin feature/your-feature-name
   ```

---

## 📋 Pull Request Guidelines

- Keep PRs focused and small — one feature/fix per PR
- Write a clear title and description
- Link any related issues
- Ensure the build passes before requesting review

---

## 🐛 Reporting Bugs

Please [open an issue](https://github.com/blazecodeprakhar/onlymedia-new/issues/new) with:
- A clear title and description
- Steps to reproduce
- Expected vs actual behavior
- Screenshots if applicable

---

## 💡 Suggesting Features

Open a [feature request issue](https://github.com/blazecodeprakhar/onlymedia-new/issues/new) with:
- A clear description of the feature
- Why it would benefit the project
- Any implementation ideas you have

---

## 📝 Code Style

- Use **TypeScript** for all new components
- Follow the existing file and folder naming conventions
- Use **Tailwind CSS** utility classes for styling
- Keep components small and focused

---

Thank you for helping make OnlyMedia better! 🎬✨
